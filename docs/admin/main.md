# 🖥 SpendScope Admin Panel (WPF)

Десктопное приложение-админка для управления системой **SpendScope**.  
Предназначено исключительно для администраторов.

Позволяет контролировать процесс распространения обновлений мобильного приложения, просматривать состояние сервера, управлять пользователями и анализировать логи.

---

## ✨ Основные возможности

- Авторизация администраторов (логин + refresh-токен)
- Управление версиями мобильного приложения
  - Загрузка новой сборки (.apk)
  - Указание версии, changelog, обязательность обновления
  - Просмотр списка всех загруженных версий
- Просмотр и очистка серверных логов
- Мониторинг состояния сервера (health-check detailed)
- Просмотр и поиск профилей пользователей
- Настройки приложения (API endpoint, таймауты, темы и др.)
- (в планах / опционально) базовая аналитика по количеству активных пользователей и установок

## 🛠 Технологический стек

| Компонент            | Технология / библиотека                                 |
|----------------------|-----------------------------------------------------------------|
| UI фреймворк         | WPF (.NET 8)                                                    |
| Архитектура          | MVVM + модульная структура (по фичам)                           |
| MVVM toolkit         | CommunityToolkit.Mvvm                                           |
| HTTP-клиент          | Refit (типобезопасные интерфейсы к API)                         |
| DI контейнер         | Microsoft.Extensions.DependencyInjection                        |
| Локальное хранилище  | Собственный сервис (json / protected storage)                  |
| Аутентификация       | JWT + refresh token (синхронизировано с backend)                |
| Навигация            | Ручная + MainWindowController                                   |
| Обработка ошибок     | Централизованный ErrorHandler + Result-паттерн                  |

## 🚀 Быстрый запуск

1. Убедитесь, что **backend** запущен и доступен
2. Откройте проект в Visual Studio
3. Укажите адрес API (можно в настройках приложения или в коде `appsettings.json`)
4. Запустите проект

```bash
dotnet run --project admin/admin.csproj
```
или просто F5 в Visual Studio.

После первого запуска рекомендуется зайти в **Настройки → API** и проверить/указать корректный адрес сервера.

# 🔐 Авторизация
Админ-панель использует те же механизмы авторизации, что и backend:

- Логин → получение access + refresh токенов
- Автоматическое обновление токена при истечении
- Принудительный выход при ошибке 401

Данные хранятся локально в зашифрованном виде (через `ProtectedData`).

# Полезные страницы и их расположение

- Логин                     → `Features/Auth/Pages/AuthLoginPage.xaml`
- Регистрация админа        → `Features/Auth/Pages/AuthRegisterPage.xaml`
- Список пользователей      → `Features/Users/UsersPage.xaml`
- Загрузка новой версии     → `Features/UploadVersion/UploadVersionPage.xaml`
- Список всех версий        → `Features/Versions/VersionsPage.xaml`
- Просмотр логов сервера    → `Features/Logs/LogsPage.xaml`
- Настройки приложения      → `Features/Settings/SettingsPage.xaml`

# ⚠️ Важные замечания

- Приложение не предназначено для конечных пользователей
- Работает только под **Windows** (WPF)
- Требует запущенного и настроенного **backend**-сервиса
- Загрузка версий требует прав администратора на сервере
